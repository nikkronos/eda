## Что это

Общая структурированная память Никиты и Богдана (брат, повар — наставник по готовке).
Телеграм-бот в **группе на троих**: оба пишут как обычно, бот слушает и раскладывает
свободный текст через LLM (Claude Haiku, structured outputs) в:

- **дневник еды** — что съедено, сытость/вкус X/5, заметки;
- **холодильник** — точный учёт остатков; списание автоматически из отчётов о еде,
  пополнение по кнопке «Куплено» из списков закупки, прямые правки текстом;
- **планы готовки** — «на завтра: …» от Богдана, читается через /today.

**Главный принцип продукта:** ноль трения. Бот никогда не требует действий — всё, что
он пишет, можно игнорировать без потери функции. Ошибся со списанием — кнопка
«Отменить». Не понял сообщение — молча сохранил в raw (kind=unparsed), поставил 🤔.
Главный риск проекта — выгорание Никиты, не техника; каждое решение проверять об это.

**Фаза 2 (НЕ в MVP):** сборка корзины Лента/Пятёрочка (браузерная автоматизация),
фото блюд в дневнике, недельный дайджест Богдану, /reparse для unparsed.

## Ключевые решения

- **Доступ:** только два telegram_id из `USERS` (.env). Группа привязывается
  автоматически первым сообщением разрешённого пользователя (или /bind); чужие
  чаты и пользователи игнорируются молча. Личка с ботом тоже работает (fallback).
- **LLM:** один вызов на сообщение; модель получает текущий холодильник и автора,
  возвращает строгий JSON (kind: meal/plan/shopping_list/inventory/chatter).
  Ошибка LLM ≠ потеря данных: всё сохраняется в raw_messages.
- **Учёт:** apply-then-correct — бот применяет списание сам и даёт «Отменить»
  (batch undo), а не спрашивает подтверждение. qty NULL = «есть, количество
  неизвестно»; NULL − x = NULL (не выдумываем точность).
- **Тишина:** chatter — ноль реакции; meal — реакция ✍; plan — 👌; текстовые
  ответы только когда изменился холодильник или нужна кнопка.
- **БД:** SQLite stdlib (без aiosqlite — 2 пользователя, нагрузки нет), времена
  в UTC, отображение через TZ_OFFSET (МСК=3).
- **LLM-провайдер:** прямой Anthropic API. OpenRouter не дешевле (цена модели та же
  + ~5% комиссия пополнения) и ломает structured outputs / anthropic SDK. Резерв на
  случай проблем с оплатой: переход на OpenRouter = правки только в llm.py.

## Локальный запуск и тесты

```powershell
# тесты (stdlib-only, API-ключи не нужны) — из корня проекта:
py -3 -m unittest discover -s tests -v

# запуск бота (нужен .env):
py -3 -m venv .venv
.venv\Scripts\pip install -r requirements.txt
.venv\Scripts\python -m bot.main
```

Windows-питон вызывать из PowerShell (`py -3`), не из git-bash.

## Деплой (когда придёт время)

Fornex, паттерн как у остальных ботов: venv + systemd-юнит `eda-bot.service`
(заготовка в `deploy/`). Перед деплоем: свежий токен из BotFather, ключ Anthropic,
`.env` только на сервере. Бота добавить в группу **админом** (иначе он не видит
обычные сообщения; альтернатива — BotFather → /setprivacy → Disable).
